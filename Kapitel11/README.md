# Kapitel 11 - NoSQL

Dieses Repository enthält Beispielcode und Tutorials zu verschiedenen modernen Datenbanktypen. Die Beispiele dienen als Ergänzung zu Kapitel 11 und zeigen grundlegende Konzepte sowie einfache Anwendungen der jeweiligen Systeme.

## Übersicht der NoSQL-Datenbanktypen

### Key-Value-Stores
- **[Redis](key-value-stores-redis/redis-howto.md)** - In-Memory-Datenbank für schnelle Zugriffe auf Schlüssel-Wert-Paare

### Document Stores
- **[MongoDB](document-stores-mongo/mongodb-howto.md)** - Dokumentenorientierte Datenbank für flexible Datenmodelle

### Graph-Datenbanken
- **[Neo4j](graph-databases-neo4j/neo4j-howto.md)** - Graph-Datenbank für Beziehungsdaten und Netzwerkanalysen
- **[GraphDB](graph-databases-graphdb/graphdb-howto.md)** - RDF-basierte Graph-Datenbank für semantische Web-Anwendungen

### Wide-Column-Stores
- **[Cassandra](wide-column-cassandra/cassandra-howto.md)** - Horizontale Skalierung für große Datenmengen

### Zeitreihendatenbanken
- **[InfluxDB](timeseries-db-influxdb/influxdb-howto.md)** - Spezialisiert auf Zeitstempeldaten und Metriken

### Vektordatenbanken
- **[Milvus](vector-databases-milvus/README.md)** - Speicherung und Suche von Vektoren für maschinelles Lernen

### Multi-Model-Datenbanken
- **[ArangoDB](multi-model-databases-arangodb/arangodb-howto.md)** - Unterstützt Dokumente, Graphen und Key-Value

### Suchmaschinen
- **[OpenSearch](suchmaschinen-opensearch/opensearch-howto.md)** - Open-Source-Suchmaschine für Volltextsuche

### Object Stores / Data Lakehouse
- **[S3 Lake (Delta Lake)](s3_lake_py/README.md)** - Objektspeicher für Data Lakehouse-Architekturen mit Delta Lake

## Quick Start

Jedes Unterkapitel enthält detaillierte Anleitungen zum Starten der jeweiligen Datenbank mit Docker oder Docker Compose sowie Beispiele zur Verwendung.
